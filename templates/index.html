🎯 ¿Qué es este simulador?

Este simulador muestra cómo funciona el mecanismo de forwarding en un pipeline RISC de 5 etapas (FI, DI, EX, MEM, WB). El forwarding permite resolver riesgos de datos (RAW hazards) sin necesidad de insertar burbujas (stalls), mejorando el rendimiento del procesador.

Formato
Ejemplo

ADD Rd, Rs, Rt
ADD R1, R2, R3

SUB R4, R1, R5
R4 = R1 - R5

LW Rd, offset(Rs)
LW R1, 0(R2)

⚙️ Configuración Inicial

  • Instrucciones: Escribe una instrucción por línea. Se ejecutarán en orden.
  • Registros iniciales: Opcional. Formato: R2=10, R3=5, R5=2
  • Habilitar Forwarding: Activa/desactiva el mecanismo de adelanto de datos.

📊 Leyenda de Estadísticas

  • Ciclos Totales: Número total de ciclos para ejecutar todas las instrucciones.
  • Stalls / Burbujas: Ciclos donde el pipeline se detuvo (menos es mejor).
  • Eventos de Forwarding: Veces que se adelantó un dato (solo con forwarding activado).
  • Eficiencia: Porcentaje de ciclos útiles vs ciclos totales.

🔬 Conceptos Técnicos

Pipeline de 5 etapas:

  • FI: Instruction Fetch - Trae la instrucción de memoria
  • DI: Instruction Decode - Decodifica y lee registros
  • EX: Execute - Ejecuta operación en ALU
  • MEM: Memory - Accede a memoria (para LW/SW)
  • WB: Write Back - Escribe resultado en registro

RAW Hazard (Read After Write): Ocurre cuando una instrucción necesita leer un registro que aún no ha sido escrito por la instrucción anterior.

Forwarding (Adelanto de datos): Técnica que conecta la salida de etapas posteriores (EX/MEM, MEM/WB) directamente a la entrada de la ALU, evitando stalls innecesarios.
